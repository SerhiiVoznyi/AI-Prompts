# Fix all failing tests in this project.

## Rules:

- Follow existing patterns, structure, naming, and style from other test files.
- Do NOT change application/business logic if tests are correct.
- If a failure is caused by incorrect business logic:
- Do NOT modify production code.
- Add a clear comment in the test describing the issue and your findings.
- Only adjust tests when they are incorrect, outdated, or inconsistent with established patterns.
- Keep changes minimal and consistent with the codebase.

## What to do:

- Run and analyze all failing tests.
- Identify root cause for each failure.
- Fix tests where appropriate.
- Document any suspected business logic issues.

## Output:

- List of fixed tests.
- List of unresolved failures with explanations.
- Notes on any detected business logic problems.
